```powershell
|---|---|

| verbose | ``${{ inputs.verbose }}`` |

### First places to look
1. **MSVC component availability** — If the run failed at `Probe and install MSVC...`, the VC.14.39.17.9.x86.x64 component may have rotated off the `windows-2022` image. Re-dispatch with ``msvc_toolset=14.40`` (and consider ``cuda_version=12.6.1`` once MX-02/v2 lands).

3. **Upstream context** — Segfault root cause: https://github.com/abetlen/llama-cpp-python/issues/1543

### Debug env
- PATH entries: $((($env:PATH -split ';') | Select-Object -First 10) -join ' ; ')
- CONDA_PREFIX: $env:CONDA_PREFIX
- Full env snapshot (first 40 entries, filtered): $(Get-ChildItem env: | Where-Object { $_.Name -match '^(CUDA|VS|PATH|CONDA|GITHUB)' } | Select-Object Name,Value | Format-Table | Out-String)
"@
$hint | Out-File -FilePath $env:GITHUB_STEP_SUMMARY -Append -Encoding utf8
Write-Host $hint
```

**Critical details:**
- `$ErrorActionPreference = 'SilentlyContinue'` is deliberate on THIS step only — the remediation-hint step running on a failed job must NOT itself throw (it would mask the real failure). This is the ONE exception to the "strict-mode + Stop everywhere" rule, and it's called out in the comment above the pragma.
- `if: failure()` ensures this step runs ONLY when an earlier step failed. `if: always()` would run on green too, polluting the summary.
- The debug-env section uses `Where-Object { $_.Name -match '^(CUDA|VS|PATH|CONDA|GITHUB)' }` to redact irrelevant env vars while surfacing the ones relevant to toolchain debugging. Do NOT dump all env vars — runner env includes credentials-ish tokens.

**What to AVOID in this task:**
- Do NOT use `if: always()` on the forensics summary step — it would emit "Preflight Green" on red runs (the most confusing possible signal).
- Do NOT use `if: always()` on the remediation-hint step either — `if: failure()` correctly scopes it.
- Do NOT add any step that mutates workflow state after the summary step. Summary/hint steps are strictly write-only.
- Do NOT reference step outputs (`${{ steps.*.outputs.* }}`) in the remediation-hint step — those outputs may be undefined (the step that would have set them failed). Using `${{ inputs.* }}` only is safe.
- Do NOT add a `timeout-minutes:` smaller than 5 on these steps. They should complete in seconds but a 1-min timeout risks flakiness on slow summary writes.
