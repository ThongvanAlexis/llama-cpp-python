Insert 3 new steps AFTER the build step and BEFORE the mamba cache save step (which has `if: always()`).

**New Step G: Assert wheel tag (BLD-10)**
- `name: 'Build: Assert wheel tag'`
- `timeout-minutes: 2`
- `shell: pwsh`
- Strict prelude
- Derive expected Python tag from dispatch input: `$pyVer = '${{ inputs.python_version }}' -replace '\.', ''` (e.g. '3.11' -> '311')
- Expected regex pattern: `"llama_cpp_python-.*-cp${pyVer}-cp${pyVer}-win_amd64\.whl$"`
- Assert exactly 1 wheel in `dist/`; throw if 0 or >1
- Assert wheel filename matches pattern; throw with remediation if not:
  - If tag is `py3-none`: "set SKBUILD_WHEEL_PY_API='' (empty string) before build"
  - If tag is `abi3`: "check pyproject.toml wheel.py-api setting"
  - If tag is `none-any`: "a platform-specific library is missing from the wheel"
- On success: `Write-Host "OK: wheel tag matches -- $wheelName"`
- Use the exact pattern from RESEARCH.md Wheel Tag Assertion code example

**New Step H: Assert wheel size (BLD-11)**
- `name: 'Build: Assert wheel size'`
- `timeout-minutes: 2`
- `shell: pwsh`
- Strict prelude
- Max size: 400 MB
- Get wheel size: `$wheel = (Get-ChildItem dist/*.whl | Select-Object -First 1)` + `$sizeMB = [math]::Round($wheel.Length / 1MB, 2)`
- Assert `$sizeMB -lt $maxSizeMB`; throw with remediation if exceeded:
  - "Reduce CMAKE_CUDA_ARCHITECTURES (currently 5 targets)"
  - "Drop a real target (e.g. 80-real if Ampere is not needed)"
  - "Check if debug symbols are accidentally embedded"
- On success: `Write-Host "OK: wheel size $sizeMB MB < $maxSizeMB MB -- $($wheel.Name)"`
- Use the exact pattern from RESEARCH.md Wheel Size Assertion code example

**New Step I: Upload wheel artifact (BLD-13)**
- `name: 'Build: Upload wheel artifact'`
- `timeout-minutes: 10`
- Use `actions/upload-artifact@v4`
- `with:`
  - name: `cuda-wheel` (this name is the contract for Phase 3 smoke-test download-artifact)
  - path: `dist/*.whl`
  - if-no-files-found: error
  - compression-level: 0 (wheel is already compressed)
  - retention-days: 7 (sufficient for Phase 3 consumption; reduce runner storage cost)

Also update the forensics summary step to include build-specific fingerprints:
- Add wheel version (from build-wheel step output): `| Wheel version | ${{ steps.build-wheel.outputs.wheel_version }} |`
- Add wheel filename: `| Wheel filename | ${{ steps.build-wheel.outputs.wheel_name }} |`
- These go in the existing markdown table (append rows)
- Guard with `${{ steps.build-wheel.outcome == 'success' }}` or just let empty values render if build step failed (the forensics step only runs on success() anyway)
